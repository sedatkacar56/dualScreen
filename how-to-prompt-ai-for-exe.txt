HOW TO PROMPT AI TO CONVERT ANY WEB APP INTO A DESKTOP .EXE
============================================================

Use these prompts step by step with Claude, ChatGPT, or any AI assistant.


STEP 1 — BUILD THE APP
-----------------------
Prompt:

"I have a web application with these files: [list your files — e.g., index.html, style.css, app.js].
Wrap it in Electron so it runs as a desktop app.
Create the main.js (Electron main process) and package.json.
Use electron with nodeIntegration enabled."


STEP 2 — TEST IT
-----------------
Prompt:

"Set up package.json so I can run it with 'npm start'. Test that it works."


STEP 3 — BUILD AS PORTABLE .EXE (no install needed)
-----------------------------------------------------
Prompt:

"Add electron-builder to the project. Configure it to build a portable .exe
for Windows. No code signing needed. The exe should run without installation —
just download and double-click. Skip code signing, I don't have a certificate."

This creates a single .exe file that runs from anywhere, leaves no traces.


STEP 4 — BUILD AS INSTALLER .EXE (like regular apps)
------------------------------------------------------
Prompt:

"Configure electron-builder to build an NSIS installer for Windows.
It should:
- NOT require administrator privileges (perMachine: false, allowElevation: false)
- Install to the user's AppData folder
- Ask the user where to install (allowToChangeInstallationDirectory: true)
- Show a progress bar during installation
- Create a desktop shortcut
- Create a Start Menu shortcut
- Include an uninstaller in Control Panel
- Offer to run the app after installation finishes
- Use oneClick: false so the full wizard is shown
- Skip code signing (signAndEditExecutable: false, signingHashAlgorithms: null)"

Key package.json settings for this:

  "nsis": {
    "oneClick": false,
    "perMachine": false,
    "allowToChangeInstallationDirectory": true,
    "allowElevation": false,
    "createDesktopShortcut": "always",
    "createStartMenuShortcut": true,
    "runAfterFinish": true
  }

IMPORTANT: If you use "perMachine": true, it will ask for admin privileges.
Use "perMachine": false so anyone can install without admin rights.


STEP 5 — BUILD BOTH VERSIONS AT ONCE
--------------------------------------
Prompt:

"Set up npm scripts so I can build both portable and installer at once.
Add these scripts:
- build:portable — builds only the portable exe
- build:installer — builds only the installer exe
- build:all — builds both"

In package.json:
  "scripts": {
    "build:portable": "electron-builder --win --config.win.target=portable",
    "build:installer": "electron-builder --win --config.win.target=nsis",
    "build:all": "electron-builder --win"
  }

And set win.target to ["portable", "nsis"] in the build config.


STEP 6 — ADD AN APP ICON
--------------------------
Prompt:

"Add a custom app icon to the project. Create a 256x256 icon with [describe
what you want — e.g., two monitors side by side with a sync symbol].
Generate both .png and .ico formats. Use it for the exe, the taskbar,
the installer, and the uninstaller."

Key settings:
  "build": {
    "icon": "icon.ico",
    "win": {
      "icon": "icon.ico"
    },
    "nsis": {
      "installerIcon": "icon.ico",
      "uninstallerIcon": "icon.ico"
    }
  }

Note: Windows requires .ico format. You can convert from .png using
the 'png-to-ico' npm package.


STEP 7 — ADD AUTO-UPDATE
--------------------------
Prompt:

"Add auto-update support using electron-updater. When I publish a new
GitHub Release, the app should:
- Check for updates on launch
- Show a dialog asking if the user wants to download
- Show another dialog when download is complete asking to restart
- Don't auto-download, let the user choose"

Key code in main.js:

  const { autoUpdater } = require('electron-updater');
  autoUpdater.autoDownload = false;
  autoUpdater.checkForUpdates();

  autoUpdater.on('update-available', () => {
    // show dialog: "Update available, download now?"
  });
  autoUpdater.on('update-downloaded', () => {
    // show dialog: "Restart to install?"
    autoUpdater.quitAndInstall();
  });

And in package.json build config:
  "publish": {
    "provider": "github",
    "owner": "your-github-username",
    "repo": "your-repo-name"
  }

IMPORTANT: For auto-update to work, you must publish releases on GitHub
with the same app name. The app checks GitHub Releases for new versions.


STEP 8 — PUBLISH ON GITHUB
----------------------------
Prompt:

"Push the code to GitHub at [your repo URL]. Create a GitHub Release
tagged v1.0.0 and attach both exe files (portable and installer).
Add a description with a table showing what each download option does."


STEP 9 — ADD A FILE PICKER (optional)
---------------------------------------
Prompt:

"Add a file picker window that opens when the app launches. Let the user
browse and select a file to open. Use Electron's dialog.showOpenDialog
through IPC — not the remote module (deprecated in modern Electron).
The picker should have a 'Browse' button and a 'Use Demo' button."


STEP 10 — ADD LIVE RELOAD (optional)
--------------------------------------
Prompt:

"Watch the loaded HTML file for changes. When the user edits and saves
the file in their text editor, automatically reload it in the app
without restarting. Use fs.watch or chokidar."


============================================================
GENERAL TIPS
============================================================

1. Always specify your target OS: "for Windows", "for Mac", "for Linux"

2. Specify portable vs installer:
   - Portable = single .exe, no install, runs from anywhere
   - NSIS installer = setup wizard, desktop shortcut, uninstaller

3. If code signing fails, tell the AI:
   "Skip code signing. Set signAndEditExecutable to false and
   signingHashAlgorithms to null."

4. If admin privileges are required but you don't have them:
   "Set perMachine to false and allowElevation to false so it
   installs to the user's AppData without needing admin."

5. If the exe is too large (Electron apps are ~80-150MB), tell the AI:
   "Use electron-builder's compression to reduce the exe size"

6. For non-web apps (Python, Node.js scripts, etc.), use different tools:
   - Python → PyInstaller: "Convert my Python script to .exe using PyInstaller"
   - Node.js → pkg: "Convert my Node.js app to .exe using pkg"
   - React/Vue/Angular → Electron: "Wrap my React app in Electron and build .exe"

7. Always test before building:
   "Run the app first with npm start to make sure it works, then build the exe"

8. Common build errors and what to tell the AI:
   - "Cannot create symbolic link" → "Skip code signing, I don't have admin"
   - "electron is only allowed in devDependencies" → "Move electron to devDependencies"
   - "author is missed" → "Add author field to package.json"

============================================================
EXAMPLE: COMPLETE SINGLE PROMPT
============================================================

If you want to do everything in one shot, use this:

"I have an HTML/CSS/JS web app in a folder. Here are my files: [list them].

Please:
1. Wrap it in Electron (main.js + package.json)
2. Add a file picker so users can choose which file to open
3. Add a custom app icon (generate a simple one)
4. Set up electron-builder with both portable and installer targets
5. No code signing, no admin privileges needed
6. Add auto-update via GitHub Releases
7. Add desktop shortcut and Start Menu shortcut in the installer
8. Test it with npm start
9. Build both .exe files
10. Push to GitHub and create a Release with both exe files attached

My GitHub repo: [your repo URL]"
